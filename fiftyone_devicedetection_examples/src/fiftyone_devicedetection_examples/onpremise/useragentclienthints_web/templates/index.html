User-Agent Client Hints example

By default, the user-agent, sec-ch-ua and sec-ch-ua-mobile HTTP headers are sent. This means that on the first request, the server can determine the browser from sec-ch-ua while other details must be derived from the user-agent. If the server determines that the browser supports client hints, then it may request additional client hints headers by setting the Accept-CH header in the response.

Select the Make second request button below to send another request to the server. This time, any additional client hints headers that have been requested will be included as evidence.
